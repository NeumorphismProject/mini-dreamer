# 产品需求文档 (PRD)

## 1. 产品概述

### 1.1 产品名称
抽象吧桌宠 - 官方网站

### 1.2 产品定位
抽象吧桌宠是一款基于 Tauri 2 开发的 Windows 桌面宠物应用的官方网站，为用户提供产品介绍、功能展示和应用下载服务。

### 1.3 目标用户
- Windows 桌面美化爱好者
- 虚拟主播/虚拟宠物爱好者
- 追求个性化桌面体验的用户
- 对语音互动和角色养成感兴趣的用户

### 1.4 核心价值
- 展示产品独特的高自由度角色定制功能
- 提供专业的产品形象和品牌展示
- 简化用户获取应用的流程
- 建立用户信任感和专业感

### 2.2 下载功能

#### 2.2.1 功能描述
用户点击下载按钮后，系统自动调用后端 API 获取最新的安装包下载链接，然后触发浏览器下载。

#### 2.2.2 API 集成
- **接口地址**：`https://4b7svf8rpf.coze.site/run`
- **请求方法**：POST
- **请求参数**：
  ```json
  {
    "action": "get_latest_file",
    "params": {}
  }
  ```
- **请求头**：
  - Authorization: Bearer {TOKEN}
  - Content-Type: application/json

#### 2.2.3 响应处理
- 成功：从 `result.download_url` 获取下载链接，调用浏览器下载功能
- 失败：显示友好的错误提示，引导用户稍后重试

#### 2.2.4 用户体验优化
- 下载按钮添加加载状态动画
- 下载开始后显示成功提示
- 错误提示使用橙色，显示 3 秒
- 成功提示使用绿色，显示 2 秒

## 3. 设计规范

### 3.1 视觉风格
- **整体风格**：现代、专业、充满活力
- **色彩方案**：
  - 主色：科技蓝 (#3B82F6) 或游戏感紫色 (#8B5CF6)
  - 辅助色：渐变色彩，体现虚拟感
  - 背景：深色主题或浅色主题切换支持
- **字体**：
  - 标题：使用有特色的字体，避免 Inter、Roboto 等常见字体
  - 正文：清晰易读的现代字体

### 3.2 动画效果
- 页面加载时使用 staggered reveal 动画
- 卡片 hover 时有微妙的缩放和阴影变化
- 按钮点击有反馈动画
- 下载过程有加载动画

### 3.3 响应式设计
- 桌面优先设计（Desktop-first）
- 移动端适配，确保在小屏幕上良好的阅读体验
- 断点设置：
  - Mobile: < 768px
  - Tablet: 768px - 1024px
  - Desktop: > 1024px

### 3.4 图标和图片
- 使用 Lucide React 图标库
- 为核心功能配备合适的图标
- 如果有产品截图或动图，优先展示

## 4. 技术需求

### 4.1 前端技术栈
- **框架**：Next.js 16 (App Router)
- **核心**：React 19
- **语言**：TypeScript 5
- **UI 组件**：shadcn/ui (基于 Radix UI)
- **样式**：Tailwind CSS 4
- **包管理器**：pnpm（必须使用，禁止 npm 或 yarn）

### 4.2 环境变量管理
必须使用 `.env.local` 文件管理敏感信息：
- `API_BASE_URL`: API 基础地址
- `API_TOKEN`: API 访问令牌

### 4.3 Hydration 防范
- 避免在 JSX 中直接使用 `typeof window`、`Date.now()`、`Math.random()` 等
- 使用 `'use client'` 配合 `useEffect` + `useState` 确保动态内容仅在客户端渲染
- 严禁非法 HTML 嵌套（如 `<p>` 嵌套 `<div>`）

### 4.4 性能优化
- 图片优化：使用 Next.js Image 组件
- 代码分割：合理使用动态导入
- 懒加载：非首屏内容采用懒加载
- 缓存策略：合理利用浏览器缓存

## 5. 用户流程

### 5.1 首页浏览流程
1. 用户访问网站首页
2. 看到英雄区域，了解产品核心卖点
3. 向下滚动查看详细功能介绍
4. 点击下载按钮获取应用

### 5.2 下载流程
1. 用户点击"立即下载"按钮
2. 按钮显示加载动画
3. 系统调用 API 获取下载链接
4. 浏览器自动开始下载文件
5. 显示下载成功提示
