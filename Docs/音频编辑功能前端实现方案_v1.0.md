## 1. 功能概述

### 1.1 功能描述

音频编辑功能允许用户对已生成的音频文件进行编辑处理，包括音量调整、淡入淡出效果添加和格式转换。

### 1.2 核心特性

- ✅ **批量编辑**：支持一次编辑多个音频
- ✅ **迭代式编辑**：不满意可重新调整参数再次应用
- ✅ **音量调整**：0-5倍音量缩放
- ✅ **淡入淡出**：毫秒级控制的音频渐变效果
- ✅ **格式转换**：支持 MP3、WAV、AAC 格式输出
- ✅ **实时预览**：编辑结果可即时播放查看
- ✅ **独立下载**：每个编辑后的音频支持单独下载

### 1.3 用户场景

**场景1：调整音效音量**
- 用户生成了一个"炸弹爆炸"音效，但觉得音量太小
- 通过编辑功能将音量放大到1.5倍
- 预览效果满意后下载使用

**场景2：添加淡入淡出效果**
- 用户生成了一个背景音乐，但开头和结尾太突兀
- 通过编辑功能添加500ms淡入和300ms淡出效果
- 迭代调整直到满意

**场景3：批量处理**
- 用户生成了多个音效，需要统一调整音量
- 选中多个音频，设置统一参数，一次性处理

### 3.2 状态管理 (Zustand)

```typescript
// src/stores/editStore.ts
import { create } from 'zustand';

interface SelectedAudio {
  audio_url: string;
  file_name: string;
}

interface OutputAudio {
  original_file_name: string;
  original_url: string;
  audio_url: string;
  file_name: string;
  file_size: number;
  file_format: string;
  audio_duration: number;
  edit_params: {
    volume_multiplier: number;
    fade_in_ms: number;
    fade_out_ms: number;
  };
  is_edited: boolean;
  temp_id: number | null;
  created_at: string;
}

interface EditStore {
  // 弹窗状态
  isOpen: boolean;

  // 输入：选中的音频
  selectedAudios: SelectedAudio[];

  // 编辑参数
  volumeMultiplier: number;
  fadeInMs: number;
  fadeOutMs: number;
  outputFormat: 'MP3' | 'WAV' | 'AAC';

  // 输出：编辑结果
  outputAudios: OutputAudio[];

  // 请求状态
  isApplying: boolean;
  error: string | null;

  // Actions
  openDialog: (audios: SelectedAudio[]) => void;
  closeDialog: () => void;
  removeAudio: (index: number) => void;
  clearAllAudios: () => void;
  setVolume: (value: number) => void;
  setFadeIn: (value: number) => void;
  setFadeOut: (value: number) => void;
  setFormat: (format: 'MP3' | 'WAV' | 'AAC') => void;
  applyEdit: () => Promise<void>;
  clearOutput: () => void;
  reset: () => void;
}
```

### 3.3 API 接口

#### 3.3.1 请求格式

```typescript
// POST /api/proxy
interface EditAudioRequest {
  action: 'edit_audio';
  params: {
    audio_list: {
      audio_url: string;
      file_name: string;
      volume_multiplier: number;  // 0.0 - 5.0, 默认 1.0
      fade_in_ms: number;         // 毫秒, 默认 0
      fade_out_ms: number;        // 毫秒, 默认 0
      output_format: 'MP3' | 'WAV' | 'AAC';  // 默认 MP3
    }[];
  };
}
```

#### 3.3.2 响应格式

```typescript
// 成功响应
interface EditAudioSuccessResponse {
  success: true;
  data: {
    edited_audios: OutputAudio[];
  };
}

// 失败响应
interface EditAudioErrorResponse {
  success: false;
  error: string;
}
```

## 5. 交互逻辑

### 5.1 弹窗打开流程

1. 用户点击编辑按钮/批量编辑按钮
2. 调用 `editStore.openDialog(selectedAudios)`
3. 弹窗打开，参数恢复默认值
4. 显示空输出列表占位

### 5.2 应用编辑流程

1. 用户调整参数（音量/淡入/淡出/格式）
2. 点击「应用」按钮
3. 显示 loading 状态
4. 调用 `edit_audio` API
5. 成功：替换输出列表内容
6. 失败：显示错误 Toast，可重新调整

### 5.3 迭代编辑流程

1. 用户查看输出结果
2. 不满意 → 调整参数 → 点击「应用」→ 结果被覆盖
3. 重复直到满意

### 5.4 关闭弹窗流程

1. 用户点击关闭/取消/弹窗外
2. 检查是否有未保存的编辑结果
3. 如果有：显示确认对话框「编辑结果未应用，确定关闭？」
4. 用户确认 → 关闭弹窗，重置状态
5. 用户取消 → 保持弹窗

## 7. 错误处理

| 错误类型 | 用户提示 | 处理方式 |
|---------|---------|---------|
| 参数验证失败 | Toast 显示错误信息 | 保持弹窗，用户可重新调整 |
| 网络请求失败 | Toast 显示"网络错误，请重试" | 保持弹窗，可重新点击应用 |
| 音频处理失败 | Toast 显示后端返回的错误信息 | 保持弹窗，可重新调整参数 |
