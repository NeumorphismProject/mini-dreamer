## 1. 需求概述

### 1.1 功能描述

开发音频编辑接口，支持对已生成的音频文件进行以下编辑操作：

- **音量调整**：放大或缩小音量（0.7倍、2.5倍等）
- **淡入效果**：从无音量到正常音量的渐变（毫秒单位）
- **淡出效果**：从正常音量到无音量的渐变（毫秒单位）

### 1.2 核心特性

- ✅ **批量编辑**：支持一次编辑多个音频
- ✅ **智能优化**：默认值参数直接返回原音频，不进行实际编辑
- ✅ **格式转换**：支持指定输出格式（MP3、WAV、AAC）
- ✅ **数据关联**：编辑后的音频关联到原始音频，支持追溯
- ✅ **参数验证**：严格的参数验证规则，防止非法操作
- ✅ **无需撤销**：不支持撤销功能

### 1.3 用户场景

**场景1：调整音效音量**
- 用户生成了一个"炸弹爆炸"音效，但觉得音量太小
- 通过编辑接口将音量放大到2.5倍

**场景2：添加淡入淡出效果**
- 用户生成了一个背景音乐，但开头和结尾太突兀
- 通过编辑接口添加1秒淡入和0.5秒淡出效果

**场景3：批量处理**
- 用户生成了10个音效，需要统一放大1.5倍
- 通过批量编辑一次性处理所有音效

## 2. 技术架构设计

### 2.1 技术选型

| 组件 | 技术 | 说明 |
|------|------|------|
| 音频处理库 | moviepy | 已安装，支持音量调整和淡入淡出 |
| 数据库 | Supabase (PostgreSQL) | 扩展现有表结构 |
| 对象存储 | S3兼容存储 | 存储编辑后的音频文件 |
| 工作流框架 | LangGraph | 节点化设计 |

### 2.2 节点架构

```
router_node
├── generate_audio_node    # 生成音频
├── edit_audio_node        # 编辑音频（新增）
├── save_audio_node        # 保存音频
├── query_temp_audio_node  # 查询临时音频
├── query_sound_effects_node # 查询音效库
└── query_background_music_node # 查询背景音乐库
```

### 2.3 数据流向

```
用户请求 → router_node → edit_audio_node
                            ↓
                        下载音频 → 应用音效 → 上传存储 → 插入数据库 → 返回结果
```

## 3. 数据库设计

### 3.1 表结构扩展

扩展 `temp_audio_files` 表，添加以下字段：

```sql
-- 扩展 temp_audio_files 表
ALTER TABLE temp_audio_files
ADD COLUMN is_edited BOOLEAN DEFAULT FALSE,
ADD COLUMN original_audio_id INTEGER,
ADD COLUMN edit_params JSONB;

-- 创建索引优化查询
CREATE INDEX idx_temp_audio_files_is_edited ON temp_audio_files(is_edited);
CREATE INDEX idx_temp_audio_files_original_id ON temp_audio_files(original_audio_id);
```

### 3.2 字段说明

| 字段 | 类型 | 说明 | 示例 |
|------|------|------|------|
| is_edited | BOOLEAN | 是否为编辑后的音频 | TRUE/FALSE |
| original_audio_id | INTEGER | 原始音频的ID（指向最原始的音频） | 8 |
| edit_params | JSONB | 编辑参数（JSON格式） | {"volume_multiplier": 1.5, "fade_in_ms": 1000, "fade_out_ms": 500} |

### 3.3 数据关联逻辑

```
原始音频（id=8, original_audio_id=NULL）
    ↓
第一次编辑（id=15, is_edited=TRUE, original_audio_id=8）
    ↓
第二次编辑（id=22, is_edited=TRUE, original_audio_id=8）
```

**关键规则**：
- 所有编辑后的音频，`original_audio_id` 都指向最原始的音频
- 通过 `original_audio_id` 可以追溯到最原始的音频文件

## 4. 接口设计

### 4.1 接口概览

| 接口名称 | Action | 功能 |
|---------|--------|------|
| 编辑音频 | edit_audio | 批量编辑音频文件 |

### 4.2 请求参数

#### HTTP 方法
```
POST /run
```

#### 请求体

```json
{
  "action": "edit_audio",
  "params": {
    "audio_list": [
      {
        "audio_url": "https://coze-coding-project.tos.coze.site/...",
        "file_name": "se_沉重_1776181955_a4248247_clipped_1s.mp3",
        "volume_multiplier": 1.5,
        "fade_in_ms": 1000,
        "fade_out_ms": 500,
        "output_format": "MP3"
      }
    ]
  }
}
```

#### 参数详细说明

| 参数名 | 类型 | 必填 | 默认值 | 说明 |
|--------|------|------|--------|------|
| audio_url | string | 是 | - | 原始音频文件的URL（必须是对象存储中的URL） |
| file_name | string | 是 | - | 原始音频的文件名（用于关联原始音频） |
| volume_multiplier | float | 否 | 1.0 | 音量倍数（0 ≤ value ≤ 5.0），1.0表示不改变音量 |
| fade_in_ms | int | 否 | 0 | 淡入时长（毫秒，0 ≤ value ≤ 音频时长） |
| fade_out_ms | int | 否 | 0 | 淡出时长（毫秒，0 ≤ value ≤ 音频时长 - fade_in_ms） |
| output_format | string | 否 | "MP3" | 输出格式，支持：MP3、WAV、AAC |

#### 成功响应

```json
{
```

## 5. 业务逻辑流程

### 5.1 单个音频处理流程

```
1. 检查参数是否为默认值
   ├─ volume_multiplier == 1.0
   └─ 是 → 直接返回原音频URL，is_edited=false，temp_id=null
   └─ 否 → 继续处理

2. 获取原始音频信息
   └─ 从对象存储获取文件元数据
      ├─ 提取音频时长
      ├─ 提取文件大小
      └─ 提取文件格式

3. 参数验证
   ├─ volume_multiplier: 0 ≤ value ≤ 5.0
   ├─ fade_in_ms: 0 ≤ value ≤ 音频时长(毫秒)
   ├─ fade_out_ms: 0 ≤ value ≤ 音频时长(毫秒) - fade_in_ms
   └─ fade_in_ms + fade_out_ms ≤ 音频时长(毫秒)
      ├─ 不通过 → 返回错误信息，整个批量编辑失败
      └─ 通过 → 继续处理

4. 下载原始音频
   └─ 从对象存储下载到 /tmp 目录

5. 应用音效
   ├─ 加载音频: AudioFileClip(file_path)
   ├─ 调整音量: volumex(volume_multiplier)
   ├─ 应用淡入: audio_fadein(fade_in_ms / 1000)  # 暂未实现
   └─ 应用淡出: audio_fadeout(fade_out_ms / 1000)  # 暂未实现

6. 保存编辑后的音频
   └─ 保存到 /tmp 目录
   └─ 文件名: edited_{timestamp}_{unique_id}.{format}

7. 上传到对象存储
   ├─ 文件路径: audio-files/sound_effects/{YYYYMM}/ 或 audio-files/background_music/{YYYYMM}/
   └─ 文件名格式: {prefix}_{style_type}_edited_{timestamp}_{unique_id}.{format}

8. 插入数据库
   ├─ 根据 file_name 查询原始音频记录
   ├─ 获取原始音频的 original_audio_id（如果为NULL则使用其id）
   ├─ 插入新记录:
   │   ├─ is_edited = TRUE
   │   ├─ original_audio_id = 原始音频ID
   │   └─ edit_params = JSON格式编辑参数
   └─ 返回 temp_id

9. 清理临时文件
   └─ 删除 /tmp 下的临时文件
```

### 5.2 批量编辑策略

```
批量编辑处理逻辑：

1. 遍历 audio_list
   └─ 对每个音频执行上述单个音频处理流程

2. 智能优化（默认值处理）
   ├─ 参数为默认值的（volume=1.0）
   │   ├─ 标记 is_edited = false
   │   ├─ 返回原音频URL
   │   ├─ 不插入数据库记录（因为没有实际编辑）
   │   └─ temp_id = null
   └─ 参数非默认值的
      ├─ 标记 is_edited = true
      ├─ 返回新音频URL
      ├─ 插入数据库记录
      └─ 返回 temp_id

3. 错误处理
   └─ 如果任何一个音频的参数验证失败
   └─ 整个批量编辑失败
   └─ 已处理的音频不会被保存（原子性操作）
```

## 6. 参数验证规则

### 6.1 音量倍数（volume_multiplier）

| 规则 | 说明 |
|------|------|
| 类型 | float（浮点数） |
| 范围 | 0.0 ≤ value ≤ 5.0 |
| 默认值 | 1.0（不改变音量） |
| 特殊值 | 0.0 表示静音（但仍会生成文件） |

**示例**：
- ✅ 0.0（静音）
- ✅ 0.7（音量缩小到70%）
- ✅ 1.0（不改变音量）
- ✅ 1.5（音量放大到150%）
- ✅ 2.5（音量放大到250%）
- ❌ -0.5（负数，无效）
- ❌ 6.0（超出范围，无效）

### 6.2 淡入时长（fade_in_ms）

| 规则 | 说明 |
|------|------|
| 类型 | int（整数） |
| 范围 | 0 ≤ value ≤ 音频时长(毫秒) |
| 默认值 | 0（无淡入） |
| 单位 | 毫秒 |
| 状态 | 暂未实现 |

### 6.3 淡出时长（fade_out_ms）

| 规则 | 说明 |
|------|------|
| 类型 | int（整数） |
| 范围 | 0 ≤ value ≤ 音频时长(毫秒) - fade_in_ms |
| 默认值 | 0（无淡出） |
| 单位 | 毫秒 |
| 状态 | 暂未实现 |

### 6.4 输出格式（output_format）

| 支持格式 | 说明 | 编码器 |
|---------|------|--------|
| MP3 | 最常用，体积小 | libmp3lame |
| WAV | 无损，体积大 | pcm_s16le |
| AAC | 高质量，兼容性好 | aac |

## 7. 文件命名规则

### 7.1 编辑后的音频文件名格式

```
{prefix}_{style_type}_edited_{timestamp}_{unique_id}.{format}
```

### 7.2 字段说明

| 字段 | 说明 | 示例 |
|------|------|------|
| prefix | 前缀，se表示音效，bm表示背景音乐 | se |
| style_type | 风格类型（拼音） | 沉重、尖锐、轻快 |
| edited | 固定标识，表示这是编辑后的音频 | edited |
| timestamp | Unix时间戳（秒） | 1776181955 |
| unique_id | 唯一标识（8位十六进制） | a4248247 |
| format | 文件格式 | mp3 |

### 7.3 命名示例

| 原始文件名 | 编辑后文件名 | 说明 |
|-----------|-------------|------|
| se_沉重_1776181955_a4248247_clipped_1s.mp3 | se_沉重_edited_1776222913_723d59c3.mp3 | 音效，音量放大1.5倍 |

## 8. 错误处理

### 8.1 错误分类

| 错误类型 | 说明 |
|---------|------|
| 参数验证失败 | 请求参数不符合规则 |
| 音频文件不存在 | 对象存储中找不到音频文件 |
| 音频处理失败 | moviepy处理音频时出错 |
| 数据库操作失败 | 插入或查询数据库时出错 |
| 对象存储失败 | 上传文件到对象存储时出错 |

## 10. 已知限制

### 10.1 淡入淡出功能

**状态**：功能已预留，但暂未实现

**原因**：moviepy 的 `audio_fadein()` 和 `audio_fadeout()` 方法类型检查报错

**解决方案**：
- 代码中已预留 TODO 注释
- 需要进一步调研 moviepy 的正确 API
- 可以考虑使用其他音频处理库（如 pydub）

### 10.2 音频时长获取

**问题**：无法通过 HEAD 请求获取音频时长

**解决方案**：
- 下载音频文件后使用 moviepy 读取时长
- 先下载再验证参数，增加了不必要的下载
