## 文档信息
- **版本**: v1.0.0
- **创建日期**: 2025-01-20
- **功能模块**: 标签管理系统

---

## 1. 功能概述

### 1.1 功能简介
实现音频文件的标签管理功能，支持标签的增删改查，以及标签与音频文件的关联管理。

### 1.2 核心特性
- 标签的完整生命周期管理（CRUD + 软删除 + 恢复）
- 标签与音频文件的关联管理（通过file_name解耦）
- 数据库层约束保证数据完整性（3个标签限制）
- 自动清理机制（删除音频时自动解除标签关联）
- 音频转移不影响标签关联（file_name不变）

---

## 2. 需求说明

### 2.1 标签管理需求
- 标签表支持完整的增删改查操作
- 标签支持软删除和恢复
- 支持批量操作（软删除、恢复）
- 支持清理无关联的软删除标签

### 2.2 标签-音频关联需求
- 一个音频文件最多关联3个标签
- 一个标签可关联无限多个音频文件
- 通过file_name关联（解耦设计）
- 音频在3张表间转移不影响标签关联
- 音频真实删除时自动解除关联

### 2.3 业务规则
- 标签名称必须唯一
- 软删除标签只能删除没有音频关联的标签
- 清理接口只删除无关联的软删除标签（每次最多100条）
- 使用触发器自动解除已删除音频的标签关联

## 3. 数据库设计

### 3.1 标签表（tags）

#### 3.1.1 表结构
```sql
CREATE TABLE tags (
    id SERIAL PRIMARY KEY,
    tag_name VARCHAR(100) NOT NULL UNIQUE,
    description TEXT,
    is_deleted BOOLEAN DEFAULT FALSE,
    deleted_at TIMESTAMP,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);
```

#### 3.1.2 字段说明
| 字段名 | 类型 | 说明 | 约束 |
|-------|------|------|------|
| id | SERIAL | 主键 | PRIMARY KEY |
| tag_name | VARCHAR(100) | 标签名称 | NOT NULL, UNIQUE |
| description | TEXT | 标签描述 | 可空 |
| is_deleted | BOOLEAN | 是否软删除 | DEFAULT FALSE |
| deleted_at | TIMESTAMP | 软删除时间 | 可空 |
| created_at | TIMESTAMP | 创建时间 | DEFAULT NOW() |
| updated_at | TIMESTAMP | 更新时间 | DEFAULT NOW() |

#### 3.1.3 索引
```sql
-- 是否删除索引
CREATE INDEX idx_tags_is_deleted ON tags(is_deleted);

-- 更新时间索引（用于排序）
CREATE INDEX idx_tags_updated_at ON tags(updated_at DESC);
```

#### 3.1.4 触发器
```sql
-- updated_at自动更新触发器
CREATE FUNCTION update_updated_at_column() RETURNS TRIGGER AS $$
BEGIN
    NEW.updated_at = CURRENT_TIMESTAMP;
    RETURN NEW;
END;
$$ LANGUAGE plpgsql;

CREATE TRIGGER update_tags_updated_at
BEFORE UPDATE ON tags
FOR EACH ROW
EXECUTE FUNCTION update_updated_at_column();
```

### 3.2 标签-音频关联表（tag_audio_mappings）

#### 3.2.1 表结构
```sql
CREATE TABLE tag_audio_mappings (
    id SERIAL PRIMARY KEY,
    file_name VARCHAR(255) NOT NULL,
    tag_id INT NOT NULL,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,

    -- 确保一个音频文件的某个标签只关联一次
    UNIQUE(file_name, tag_id),

    -- 外键约束：标签必须存在，删除标签时级联删除关联
    CONSTRAINT fk_tag FOREIGN KEY (tag_id) REFERENCES tags(id) ON DELETE CASCADE
);
```

#### 3.2.2 字段说明
| 字段名 | 类型 | 说明 | 约束 |
|-------|------|------|------|
| id | SERIAL | 主键 | PRIMARY KEY |
| file_name | VARCHAR(255) | 文件名（关联键） | NOT NULL |
| tag_id | INT | 标签ID | NOT NULL, FOREIGN KEY |
| created_at | TIMESTAMP | 创建时间 | DEFAULT NOW() |

#### 3.2.3 约束说明
- **UNIQUE(file_name, tag_id)**: 确保一个音频文件的某个标签只关联一次
- **FOREIGN KEY**: 标签必须存在，删除标签时级联删除关联

#### 3.2.4 索引
```sql
-- 文件名索引（查询音频的标签）
CREATE INDEX idx_tag_audio_mappings_file_name ON tag_audio_mappings(file_name);

-- 标签ID索引（查询标签的音频）
CREATE INDEX idx_tag_audio_mappings_tag_id ON tag_audio_mappings(tag_id);
```

### 3.3 触发器：删除音频时自动解除关联

#### 3.3.1 触发器函数
```sql
-- 删除音频时自动解除标签关联的函数
CREATE FUNCTION delete_audio_tags() RETURNS TRIGGER AS $$
BEGIN
    DELETE FROM tag_audio_mappings WHERE file_name = OLD.file_name;
    RETURN NULL;
END;
$$ LANGUAGE plpgsql;
```

## 4. 存储过程设计

### 4.1 3个标签限制检查

#### 4.1.1 检查函数
```sql
CREATE OR REPLACE FUNCTION check_audio_tag_limit(p_file_name VARCHAR, p_tag_id INT)
RETURNS BOOLEAN AS $$
DECLARE
    v_count INT;
BEGIN
    -- 检查该音频文件已关联的标签数量
    SELECT COUNT(*) INTO v_count
    FROM tag_audio_mappings
    WHERE file_name = p_file_name;

    -- 如果已达到3个，返回false
    IF v_count >= 3 THEN
        RETURN FALSE;
    END IF;

    -- 检查是否已经关联了这个标签
    SELECT COUNT(*) INTO v_count
    FROM tag_audio_mappings
    WHERE file_name = p_file_name AND tag_id = p_tag_id;

    IF v_count > 0 THEN
        RETURN FALSE;  -- 已关联，不允许重复
    END IF;

    RETURN TRUE;
END;
$$ LANGUAGE plpgsql;
```

#### 4.1.2 安全添加标签
```sql
CREATE OR REPLACE FUNCTION safe_add_audio_tag(p_file_name VARCHAR, p_tag_id INT)
RETURNS TABLE(success BOOLEAN, message TEXT) AS $$
DECLARE
    v_can_add BOOLEAN;
BEGIN
    -- 检查是否可以添加
    SELECT check_audio_tag_limit(p_file_name, p_tag_id) INTO v_can_add;

    IF NOT v_can_add THEN
        -- 检查是否已达到3个标签
        DECLARE
            v_current_count INT;
        BEGIN
            SELECT COUNT(*) INTO v_current_count
            FROM tag_audio_mappings
            WHERE file_name = p_file_name;

            IF v_current_count >= 3 THEN
                RETURN QUERY SELECT FALSE, '该音频文件已达到3个标签上限'::TEXT;
            ELSE
                RETURN QUERY SELECT FALSE, '该音频文件已关联此标签'::TEXT;
            END IF;
        END;
    ELSE
        -- 添加关联
        BEGIN
            INSERT INTO tag_audio_mappings (file_name, tag_id)
            VALUES (p_file_name, p_tag_id);

            RETURN QUERY SELECT TRUE, '添加标签成功'::TEXT;
        EXCEPTION
            WHEN OTHERS THEN
                RETURN QUERY SELECT FALSE, SQLERRM::TEXT;
        END;
    END IF;
END;
$$ LANGUAGE plpgsql;
```

### 4.2 批量软删除标签

#### 4.2.1 存储过程
```sql
CREATE OR REPLACE FUNCTION soft_delete_tags(p_ids INT[])
RETURNS TABLE(success BOOLEAN, message TEXT, total_count INT, deleted_count INT, failed_count INT, details JSON)
LANGUAGE plpgsql
AS $$
DECLARE
    v_record RECORD;
    v_count INT := 0;
    v_deleted INT := 0;
    v_failed INT := 0;
    v_details JSON := '[]'::json;
    v_has_mapping BOOLEAN;
BEGIN
    -- 逐个处理每个ID
    FOR v_record IN SELECT unnest(p_ids) AS id LOOP
        BEGIN
            v_count := v_count + 1;

            -- 检查标签是否存在且未删除
            IF NOT EXISTS (
                SELECT 1 FROM tags
                WHERE id = v_record.id AND is_deleted = FALSE
            ) THEN
                v_failed := v_failed + 1;
                v_details := v_details || json_build_object('id', v_record.id, 'error', '标签不存在或已删除')::jsonb;
                CONTINUE;
            END IF;

            -- 检查是否有音频关联
            SELECT EXISTS (
                SELECT 1 FROM tag_audio_mappings
                WHERE tag_id = v_record.id
            ) INTO v_has_mapping;

            IF v_has_mapping THEN
                v_failed := v_failed + 1;
                v_details := v_details || json_build_object('id', v_record.id, 'error', '标签存在关联，无法删除')::jsonb;
                CONTINUE;
            END IF;

            -- 软删除标签
            UPDATE tags
            SET is_deleted = TRUE,
                deleted_at = CURRENT_TIMESTAMP
            WHERE id = v_record.id;

            v_deleted := v_deleted + 1;

        EXCEPTION
            WHEN OTHERS THEN
                v_failed := v_failed + 1;
                v_details := v_details || json_build_object('id', v_record.id, 'error', SQLERRM)::jsonb;
        END;
    END LOOP;

    RETURN QUERY SELECT
        (v_failed = 0) AS success,
        CASE WHEN v_failed = 0 THEN '成功删除所有标签' ELSE ('部分失败，成功 ' || v_deleted || '，失败 ' || v_failed) END AS message,
        v_count AS total_count,
        v_deleted AS deleted_count,
        v_failed AS failed_count,
        v_details AS details;
END;
$$;
```

#### 4.2.2 业务规则
- 只能删除未软删除的标签
- 只能删除没有任何音频关联的标签
- 逐个处理，单个失败不影响整体
- 返回详细的处理结果

### 4.3 批量恢复标签

#### 4.3.1 存储过程
```sql
CREATE OR REPLACE FUNCTION restore_tags(p_ids INT[])
RETURNS TABLE(success BOOLEAN, message TEXT, total_count INT, restored_count INT, failed_count INT, details JSON)
LANGUAGE plpgsql
AS $$
DECLARE
    v_record RECORD;
    v_count INT := 0;
    v_restored INT := 0;
    v_failed INT := 0;
    v_details JSON := '[]'::json;
BEGIN
    -- 逐个处理每个ID
    FOR v_record IN SELECT unnest(p_ids) AS id LOOP
        BEGIN
            v_count := v_count + 1;

            -- 检查标签是否存在且已删除
            IF NOT EXISTS (
                SELECT 1 FROM tags
                WHERE id = v_record.id AND is_deleted = TRUE
            ) THEN
                v_failed := v_failed + 1;
                v_details := v_details || json_build_object('id', v_record.id, 'error', '标签不存在或未删除')::jsonb;
                CONTINUE;
            END IF;

            -- 恢复标签
            UPDATE tags
            SET is_deleted = FALSE,
                deleted_at = NULL
            WHERE id = v_record.id;

            v_restored := v_restored + 1;

        EXCEPTION
            WHEN OTHERS THEN
                v_failed := v_failed + 1;
                v_details := v_details || json_build_object('id', v_record.id, 'error', SQLERRM)::jsonb;
        END;
    END LOOP;

    RETURN QUERY SELECT
        (v_failed = 0) AS success,
        CASE WHEN v_failed = 0 THEN '成功恢复所有标签' ELSE ('部分失败，成功 ' || v_restored || '，失败 ' || v_failed) END AS message,
        v_count AS total_count,
        v_restored AS restored_count,
        v_failed AS failed_count,
        v_details AS details;
END;
$$;
```

#### 4.3.2 业务规则
- 只能恢复已软删除的标签
- 逐个处理，单个失败不影响整体
- 返回详细的处理结果

### 4.4 清理软删除标签

#### 4.4.1 存储过程
```sql
CREATE OR REPLACE FUNCTION cleanup_deleted_tags()
RETURNS TABLE(success BOOLEAN, message TEXT, deleted_count INT) AS $$
DECLARE
    v_deleted_count INT := 0;
BEGIN
    -- 删除没有音频关联的软删除标签，最多100条
    DELETE FROM tags
    WHERE is_deleted = TRUE
    AND id NOT IN (SELECT DISTINCT tag_id FROM tag_audio_mappings)
    LIMIT 100;

    GET DIAGNOSTICS v_deleted_count = ROW_COUNT;

    RETURN QUERY SELECT
        TRUE AS success,
        '成功清理 ' || v_deleted_count || ' 条软删除标签'::TEXT AS message,
        v_deleted_count AS deleted_count;
END;
$$ LANGUAGE plpgsql;
```

#### 4.4.2 业务规则
- 只删除没有音频关联的软删除标签
- 每次最多删除100条
- 返回实际删除数量

## 6. 数据流转设计

### 6.1 音频转移流程

```
场景1: 音频从temp_audio_files转移到sound_effects

步骤:
1. 删除temp_audio_files中的记录（id=1, file_name="a.mp3"）
2. → 触发器自动删除tag_audio_mappings中的关联（file_name="a.mp3"）
3. 插入到sound_effects（id=100, file_name="a.mp3"）
4. 重新添加标签关联（通过file_name="a.mp3"）

结果: 标签关联保持不变（因为file_name相同）
```

```
场景2: 音频在sound_effects和premium_sound_effects之间转移

步骤:
1. 从sound_effects删除记录（id=100, file_name="a.mp3"）
2. → 触发器自动删除tag_audio_mappings中的关联
3. 插入到premium_sound_effects（id=200, file_name="a.mp3"）
4. 重新添加标签关联（通过file_name="a.mp3"）

结果: 标签关联保持不变（因为file_name相同）
```

### 6.2 音频真实删除流程

```
场景: 音频被永久删除

步骤:
1. 从sound_effects删除记录（id=100, file_name="a.mp3"）
2. → 触发器自动删除tag_audio_mappings中的关联（file_name="a.mp3"）

结果: 标签关联自动解除，标签本身保留
```

### 6.3 标签删除流程

```
场景: 删除标签

步骤:
1. 调用soft_delete_tags，传入标签ID
2. 检查标签是否有音频关联
3. 如果有关联 → 拒绝删除
4. 如果没有关联 → 软删除标签

结果: 只有无关联的标签可以被删除
```

## 7. 关键设计说明

### 7.1 解耦设计

#### 设计目标
音频文件在3张表间转移时，标签关联不受影响

#### 解决方案
- 通过`file_name`而非`id`关联音频文件和标签
- `file_name`是音频文件的全局唯一标识
- 音频转移时ID会变化，但`file_name`不变

#### 优势
- 音频转移无需迁移标签关联数据
- 查询简单直接（通过file_name）
- 扩展性好（未来新增音频表无需修改关联表）

---

### 7.2 数据完整性保证

#### 7.2.1 数据库层约束
- **标签名称唯一**: `UNIQUE(tag_name)`
- **标签关联唯一**: `UNIQUE(file_name, tag_id)`
- **外键约束**: `FOREIGN KEY(tag_id) REFERENCES tags(id)`

#### 7.2.2 数据库层业务规则
- **3个标签限制**: 通过存储过程`check_audio_tag_limit`实现
- **软删除约束**: 只允许删除无关联的标签（存储过程检查）

#### 7.2.3 自动清理机制
- **触发器**: 删除音频时自动解除标签关联
- **级联删除**: 删除标签时级联删除关联

---

### 7.3 性能优化

#### 7.3.1 索引设计
- `idx_tags_is_deleted`: 加速是否删除查询
- `idx_tags_updated_at`: 加速排序查询
- `idx_tag_audio_mappings_file_name`: 加速通过file_name查询标签
- `idx_tag_audio_mappings_tag_id`: 加速通过tag_id查询音频

#### 7.3.2 分页查询
- 所有列表查询支持分页
- 避免一次性加载大量数据

#### 7.3.3 清理限制
- 清理接口每次最多删除100条
- 避免长时间锁定表

## 8. 异常处理

### 8.1 标签名称重复
- **场景**: 创建或更新标签时，标签名称已存在
- **处理**: 返回失败，提示标签名称已存在

### 8.2 标签达到3个限制
- **场景**: 为音频添加标签时，已达到3个标签
- **处理**: 返回失败，提示"该音频文件已达到3个标签上限"

### 8.3 标签存在关联
- **场景**: 尝试软删除有音频关联的标签
- **处理**: 返回失败，提示"标签存在关联，无法删除"

### 8.4 标签不存在
- **场景**: 操作不存在的标签ID
- **处理**: 返回失败，提示"标签不存在"

### 8.5 重复添加标签
- **场景**: 为音频添加已关联的标签
- **处理**: 返回失败，提示"该音频文件已关联此标签"
