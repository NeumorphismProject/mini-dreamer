# 标签功能前端实现设计方案

## 文档信息
- **版本**: v1.0.0
- **创建日期**: 2026-04-16
- **功能模块**: 标签管理系统前端实现

---

## 1. 需求概述

### 1.1 核心功能
1. **标签管理页面** - 独立的标签管理列表页，支持标签的增删改查
2. **标签筛选区域** - 三个列表页（音效库、精品音效库、临时音频）公用的标签筛选组件
3. **保存音频弹窗增强** - SaveModal 增加标签选择功能
4. **批量添加标签** - 为音频文件批量添加标签

### 1.2 业务规则（来自设计文档）
- 一个音频文件最多关联 **3个标签**
- 一个标签可关联无限多个音频文件
- 标签通过 `file_name` 与音频关联（跨3张表）
- 音频在3张表间转移不影响标签关联

## 2. API 接口分析

### 2.1 标签管理接口（7个）

| 接口 | 功能 | 用途 |
|------|------|------|
| `query_tags` | 分页查询标签 | 列表展示、筛选 |
| `create_tag` | 创建标签 | 新增标签 |
| `update_tag` | 更新标签 | 编辑标签 |
| `soft_delete_tags` | 批量软删除 | 删除标签 |
| `query_deleted_tags` | 查询软删除标签 | 回收站 |
| `restore_tags` | 批量恢复标签 | 恢复标签 |
| `cleanup_deleted_tags` | 清理无关联软删除标签 | 清理 |

### 2.2 标签-音频关联接口（3个）

| 接口 | 功能 | 用途 |
|------|------|------|
| `query_audio_tags` | 查询音频标签 | 获取音频关联的标签 |
| `add_audio_tags_batch` | 批量添加音频标签 | 批量添加标签 |
| `delete_audio_tag` | 删除音频标签 | 移除单个标签关联 |

### 2.3 列表查询接口（已支持 tag_ids）

| 接口 | tag_ids 参数 | 说明 |
|------|-------------|------|
| `query_sound_effects` | tag_ids: "1,2,3" | 音效库，支持多标签OR查询 |
| `query_premium_sound_effects` | tag_ids: "1,2,3" | 精品音效库，支持多标签OR查询 |
| `query_temp_audio` | tag_ids: "1,2,3" | 临时音频，支持多标签OR查询 |

**重要：响应中包含 tags 字段**

每个音频项的响应中都包含 `tags` 字段，用于展示该音频关联的标签：

```json
{
  "list": [
    {
      "id": 15,
      "file_name": "se_清脆_1776167871_3a30fb8e.wav",
      "file_format": "WAV",
      "audio_url": "https://example.com/audio/file.wav",
      "audio_duration": 2,
      "file_size": 176444,
      "style_type": "清脆",
      "description": "清脆的水滴声",
      "created_at": "2026-04-15T17:00:00.000000+08:00",
      "updated_at": "2026-04-15T17:00:00.000000+08:00",
      "tags": [
        {"id": 1, "tag_name": "标签1"},
        {"id": 2, "tag_name": "标签2"}
      ]
    }
  ]
}
```

**tags 字段规则**：
- `tags` 可能为 `null`、空数组 `[]` 或包含标签对象
- 如果 `tags` 为 `null` 或空数组 → 该音频没有标签，不展示
- 如果 `tags` 有值 → 在 AudioCard 中展示标签 Chip

### 2.4 保存音频接口（已支持 tag_ids）

| 接口 | tag_ids 参数 | 说明 |
|------|-------------|------|
| `save_audio` | tag_ids: "1,2,3" | 保存时同时添加标签关联 |

## 4. API 服务层

### 4.1 新增 API 文件

```typescript
// src/services/api/tag-api.ts

import { apiClient } from './client';
import type {
  TagItem,
  QueryTagsParams,
  QueryTagsResponse,
  CreateTagParams,
  UpdateTagParams,
  QueryAudioTagsResponse,
  AddAudioTagsBatchResponse,
} from '@/types';

export const tagApi = {
  // ============ 标签管理 ============

  // 查询标签列表
  queryTags: (params?: QueryTagsParams): Promise<QueryTagsResponse> =>
    apiClient.request<QueryTagsResponse>('query_tags', params || {}),

  // 创建标签
  createTag: (params: CreateTagParams): Promise<TagItem> =>
    apiClient.request<TagItem>('create_tag', params),

  // 更新标签
  updateTag: (params: UpdateTagParams): Promise<TagItem> =>
    apiClient.request<TagItem>('update_tag', params),

  // 批量软删除标签
  softDeleteTags: (ids: number[]): Promise<{
    success: boolean;
    message: string;
    total_count: number;
    deleted_count: number;
    failed_count: number;
    details: { id: number; error?: string }[];
  }> =>
    apiClient.request('soft_delete_tags', { ids }),

  // 查询软删除标签
  queryDeletedTags: (params?: QueryTagsParams): Promise<QueryTagsResponse> =>
    apiClient.request<QueryTagsResponse>('query_deleted_tags', params || {}),

  // 批量恢复标签
  restoreTags: (ids: number[]): Promise<{
    success: boolean;
    message: string;
    total_count: number;
    restored_count: number;
    failed_count: number;
    details: { id: number; error?: string }[];
  }> =>
    apiClient.request('restore_tags', { ids }),

  // 清理软删除标签
  cleanupDeletedTags: (): Promise<{
    success: boolean;
    message: string;
    deleted_count: number;
  }> =>
    apiClient.request('cleanup_deleted_tags', {}),

  // ============ 标签-音频关联 ============

  // 查询音频标签
  queryAudioTags: (fileName: string): Promise<QueryAudioTagsResponse> =>
    apiClient.request<QueryAudioTagsResponse>('query_audio_tags', { file_name: fileName }),

  // 批量添加音频标签
  addAudioTagsBatch: (fileName: string, tagIds: number[]): Promise<AddAudioTagsBatchResponse> =>
    apiClient.request<AddAudioTagsBatchResponse>('add_audio_tags_batch', {
      file_name: fileName,
      tag_ids: tagIds.join(','),
    }),

  // 删除音频标签
  deleteAudioTag: (fileName: string, tagId: number): Promise<{ message: string }> =>
    apiClient.request('delete_audio_tag', { file_name: fileName, tag_id: tagId }),
};
```

## 5. Store 设计

### 5.1 标签 Store

```typescript
// src/stores/tag-store.ts

import { create } from 'zustand';
import type { TagItem } from '@/types';
import { tagApi } from '@/services/api';

interface TagState {
  // 标签列表
  tags: TagItem[];
  total: number;
  page: number;
  pageSize: number;
  isLoading: boolean;

  // 选中的标签（用于筛选）
  selectedTagIds: number[];

  // 软删除标签
  deletedTags: TagItem[];
  deletedTotal: number;
  deletedPage: number;

  // Actions
  fetchTags: (params?: { page?: number; tag_name?: string }) => void;
  fetchMoreTags: () => void;
  setSelectedTagIds: (ids: number[]) => void;
  toggleTagId: (id: number) => void;
  clearSelectedTags: () => void;
  fetchDeletedTags: (params?: { page?: number; tag_name?: string }) => void;
  reset: () => void;
}

export const useTagStore = create<TagState>((set, get) => ({
  tags: [],
  total: 0,
  page: 1,
  pageSize: 20,
  isLoading: false,
  selectedTagIds: [],
  deletedTags: [],
  deletedTotal: 0,
  deletedPage: 1,

  fetchTags: async (params) => {
    set({ isLoading: true });
    try {
      const { page, pageSize, selectedTagIds } = get();
      const result = await tagApi.queryTags({
        page: params?.page || page,
        page_size: pageSize,
        tag_name: params?.tag_name,
      });
      set({
        tags: result.list,
        total: result.total,
        page: result.page,
        isLoading: false,
      });
    } catch (error) {
      console.error('Failed to fetch tags:', error);
      set({ isLoading: false });
    }
  },

  fetchMoreTags: async () => {
    const { page, pageSize, tags, isLoading } = get();
    if (isLoading) return;

    const nextPage = page + 1;
    try {
      const result = await tagApi.queryTags({
        page: nextPage,
        page_size: pageSize,
      });
      set({
        tags: [...tags, ...result.list],
        total: result.total,
        page: nextPage,
      });
    } catch (error) {
      console.error('Failed to fetch more tags:', error);
    }
  },

  setSelectedTagIds: (ids) => set({ selectedTagIds: ids }),

  toggleTagId: (id) => {
    const { selectedTagIds } = get();
    if (selectedTagIds.includes(id)) {
      set({ selectedTagIds: selectedTagIds.filter((i) => i !== id) });
    } else {
      set({ selectedTagIds: [...selectedTagIds, id] });
    }
  },

  clearSelectedTags: () => set({ selectedTagIds: [] }),

  fetchDeletedTags: async (params) => {
    const { deletedPage } = get();
    try {
      const result = await tagApi.queryDeletedTags({
        page: params?.page || deletedPage,
        page_size: 20,
        tag_name: params?.tag_name,
      });
      set({
        deletedTags: params?.page === 1 ? result.list : [...get().deletedTags, ...result.list],
        deletedTotal: result.total,
        deletedPage: result.page,
      });
    } catch (error) {
      console.error('Failed to fetch deleted tags:', error);
    }
  },

  reset: () =>
    set({
      tags: [],
      total: 0,
      page: 1,
      selectedTagIds: [],
      deletedTags: [],
      deletedTotal: 0,
      deletedPage: 1,
    }),
}));
```

## 12. 注意事项

1. **懒加载实现**：使用 Intersection Observer 或 scroll 事件监听
2. **防抖处理**：搜索输入使用 debounce（300ms）
3. **状态同步**：列表刷新时同步 TagStore 的 tags 数组
4. **错误处理**：API 错误需要 toast 提示
5. **Hydration**：确保动态内容使用 `useEffect` + `useState`
6. **AudioCard 标签展示**：必须检查 `tags` 是否有值，避免 null/undefined 导致的渲染错误
7. **标签筛选联动**：三个列表页面共享 `useTagStore.selectedTagIds`，筛选后需要刷新列表
